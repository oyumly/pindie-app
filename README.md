# Pindie — Портал инди-игр (Фронтенд)

Это клиентская часть веб-приложения «Pindie» — портала с рейтингами инди-игр, где пользователи могут просматривать новинки, авторизовываться и голосовать за проекты. Разработано в рамках курса: [**«JavaScript от Яндекса: с нуля до веб-разработчика»**](https://practicum.yandex.ru/web/ "Курс «JavaScript от Яндекса: с нуля до веб-разработчика» — Яндекс Практикум"). 

## 🔗 Ссылки
- **Бэкенд проекта:** [pindie-backend](https://github.com/oyumly/pindie-backend)

## 🛠 Стек технологий
В отличие от ранних этапов, где использовался обычный HTML/JS, этот проект написан на современном стеке:
- **React (v18)** — основа пользовательского интерфейса
- **Next.js (v14)** — фреймворк для SSR, роутинга и сборки
- **Zustand** — легкий стейт-менеджер для управления глобальным состоянием
- **Swiper** — библиотека для создания современных слайдеров/каруселей игр
- **React Router Dom** — для дополнительной маршрутизации (если используется на клиенте)

## 🚀 Основной функционал
- Динамическая подгрузка игр с бэкенда и их распределение по категориям (Новинки, Шутеры, Пиксельные и т.д.).
- Система авторизации: всплывающие окна (попапы) для входа в аккаунт.
- Механика голосования: возможность отдать голос за любимую игру (доступно только авторизованным пользователям).
- Адаптивный интерфейс с использованием слайдеров (Swiper).

## ⚙️ Запуск проекта локально

Поскольку фронтенд взаимодействует с API, для полноценной работы **необходимо сначала запустить локальный [бэкенд](https://github.com/oyumly/pindie-backend)** (убедитесь, что сервер бэкенда работает и принимает запросы на `localhost`).

1. Склонируйте этот репозиторий:
   ```bash
   git clone https://github.com/oyumly/pindie-app.git
   ```
2. Откройте папку проекта и установите зависимости:
   ```bash
   cd pindie-app
   npm install
   ```
3. Запустите проект в режиме разработки (dev-сервер Next.js):
   ```bash
   npm run dev
   ```

После этого приложение будет доступно по адресу [http://localhost:3000](http://localhost:3000) (или на другом порту, если 3000 занят бэкендом — Next.js сам предложит порт `3001`).
